# Contributing to Syntecxhub ATM Simulation

We welcome contributions to Syntecxhub ATM Simulation! Please follow the guidelines below to ensure a smooth contribution process.

## Reporting Issues

- To report an issue, please navigate to the [Issues](https://github.com/RohancRevankar/Syntecxhub_-ATM_Simulation/issues) section of the repository.
- Describe the issue clearly, providing as much detail as possible, including steps to reproduce, expected results, and actual results.

## Submitting Pull Requests

1. **Fork the Repository**: Click on the "Fork" button on the top right corner of the repository page.
2. **Clone Your Fork**: Use the following command to clone your forked repository:
   ```bash
   git clone https://github.com/YOUR_USERNAME/Syntecxhub_-ATM_Simulation.git
   ```
3. **Create a New Branch**: It’s recommended to create a new branch for each feature or fix:
   ```bash
   git checkout -b feature-or-fix-name
   ```
4. **Make Your Changes**: Make the desired changes to the codebase.
5. **Commit Your Changes**: Ensure your commit message is clear and descriptive:
   ```bash
   git commit -m "Description of changes"
   ```
6. **Push to Your Fork**:
   ```bash
   git push origin feature-or-fix-name
   ```
7. **Create a Pull Request**: Go back to the original repository and create a pull request from your forked repository.

## Code Style Guidelines

- Please adhere to the coding standards used in the project. Be consistent with naming conventions, indentation, and formatting.
- Write clear and descriptive comments in your code.
- Ensure your code passes any existing tests before submitting a pull request.

Thank you for contributing! Your help is greatly appreciated!
